Updated: 2026-07-10 · Scope: staging only (rbdxzlzkxyprertdmpga) · NONCLINICAL

This page defines M1 and nothing else. Prior board content (north star, lanes, decision log) is retired. M1 is "done" only when the acceptance test below passes end to end, observed from deployed staging.

Goal

A Jason Yim account created by a human in the staging mobile app produces one continuous, ID-linked case:

mobile onboarding → backend ingest → synthetic lab/genetic enrichment → model runs → action candidates → real AI-drafted care-plan notes → physician review/release → mobile readback

Verified as a single lineage. No fixtures, no bearer token, no fabricated inputs.

Data contract (immutable for M1)

SourceProvides
Mobile onboardingIdentity, demographics, history, symptoms, family history, consents, preferences
HealthKitActual device data when connected
Backend synthetic enrichmentLabs and genetics only
NothingNo synthetic vitals or wearables. Missing stays missing. No representative fixture packet.

Acceptance test — the only definition of done

  1. A human creates a Jason Yim account in the staging mobile app (Apple sign-in).
  2. That account submits onboarding plus available HealthKit data.
  3. Backend attaches synthetic labs and genetics keyed to that exact patient ID.
  4. Models run: sufficient inputs compute; insufficient inputs return explicit missing_required_inputs with named blockers. No representative anchors.
  5. Action library emits provenance-linked candidates.
  6. A real LLM drafts the care-plan notes, recording model version, prompt version, inputs, output, citations, and safety checks.
  7. The case appears in the deployed tokenless staging dashboard (no login form, no bearer token).
  8. Physician reviews, signs, and releases.
  9. The same mobile account renders the released plan.
  10. One audit query shows an unbroken chain of IDs and packet hashes across all prior stages.

Backlog — why M1 is not yet done

  1. Backend synthetic-enrichment seam. Attach synthetic labs and genetics to the real mobile packet by patient ID. Does not exist as a product path today.
  2. Real AI synthesis step. Care-plan notes are currently hard-coded templates in TypeScript. Replace with a real LLM call plus recorded provenance.
  3. Publish and verify the tokenless dashboard against the deployed backend. Backend is deployed; the revised frontend is not yet published.

Out of scope for M1

  • Production project. Staging only.
  • Multi-patient or queue-at-scale behavior.
  • Real (non-synthetic) labs or genetics.
  • Clinical validity of any model output. Everything remains NONCLINICAL.
  • Payments, logistics, and messaging beyond what the release package already carries.
