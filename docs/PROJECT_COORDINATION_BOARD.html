Staging acceptance path

The board is green only when this passes end to end.
  1. Install TestFlight staging build with real API base.
  2. Create account with Apple or email.
  3. Backend creates account, auth identity, session, and patient.
  4. Complete onboarding with HealthKit allowed or denied.
  5. Backend stores onboarding steps and validates patient_packet.v1.
  6. Packet submission creates an idempotent physician work item.
  7. Physician sees queue item, starts review, creates preview, signs, and releases.
  8. Patient app polls backend and renders only release_package.v1.
  9. Backend restart does not lose state.
  10. Audit log reconstructs account, packet, review, release, and patient visibility events.

Maintenance rule: if this page disagrees with the production strategy, update both in the same change or state which one is authoritative. Do not treat this board as proof of production readiness without passing the acceptance path.
